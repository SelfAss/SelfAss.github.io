  • 版本控制工具之Git的基本使用（一）

    2016-06-19

    写在前面

    Git是Linux发明者Linus开发的一款一个分布式版本控制系统。它可以在Linux，Unix和windows平台上运行。

    那么什么又是版本控制呢？先来看看百度百科给出的定义：

    版本控制透过文档控制（documentation control）记录程序各个模组的改动，并为每次改动编上序号。
这种方法是工程图（engineering drawings）维护（maintenance）的标准做法， 它伴随着工程图从图的诞生一直到图的定型。
 一种简单的版本控制形式，例如，赋给图的初版一个版本等级“A”。
当做了第一次改变后，版本等级改为“B”，以此类推等等。

    用一个通俗的话来讲：当我们在开发一个项目的时候，为了防止代码丢失，我们肯定会想办法在本地硬盘存放一份，在服务器端也存放一份，万一本地的代码丢失了，我们还可以在从服务器中获取我们原来的代码。但是我们如何保证本地代码和服务器代码是一致的呢？再者，我们开发一个项目并不是一个人在进行开发，一个项目是由多人合作开发完成的，可能一份代码并不是由一个人在进行维护，大家都在改同一份代码，怎样能够知道这份代码哪里是由谁更改的。另外，项目开发中，难免会出现bug，如果一个项目刚上线，就出现了一个很严重的bug，这时候就需要对代码进行还原。。。还有，我们需要知道代码提交的记录以及日志信息，等等这些功能。

    上述的这些情况，就是版本控制系统能够实现的功能。当下流行的版本控制工具有很多，比如svn，git。如果在早些年，我会推荐你们用svn，但是以目前的现状来看，大部分公司都是用git来进行版本控制的，所以git是当之无愧当下最流行的版本控制系统。

    Git下载与安装

    git下载和安装就不说了，不会的自行百度。安装要提醒一下，可以安装是可以带图形界面和命令行的，但是我不推荐你们使用图形界面来操作git。只有多多使用命令行，你才能理解git每一步操作的意义。

    如何判断git是否安装成功了，可以在gitbash页面输入

    git --version

    如果能看到git的版本信息，就可以认为git是安装成功了。

    或者你也可以直接输入

    git --help

    能够看到下面的帮助信息也能说明git是安装成功了。

    git 基本配置信息

    git安装好了之后，我们在开始使用git之前，必须先对git进行配置。

    1、配置用户名和邮箱

    git config --global user.name "yourname"
git config --global user.email "youremail@163.com"

    注意：上面的指令”yourname” 填你自己的用户名，”youremail@163.com” 这里填你自己的邮箱

    2、验证用户名和邮箱
    执行完上述指令后，我们同样可以用这个指令去检测用户名和密码

    git config --global user.name
git config --global user.email

    git branch基础知识

    branch，就是分支，分流，分叉的意思，通俗来说，一个git枝干上有一个master和多个branch，默认情况下是没有branch（分支）的，只有一个主分支（master），想要查看分支信息，可以使用如下指令来进行查看：

    git branch

    分支的概念是很重要的，举个比较容易理解的例子，现在有一个项目Project，项目组长会申请一个master，然后交由下面的几个人进行开发，甲开发的是A模块，那么甲应该在主分支的基础上新建一个分支a，在分支a上进行开发，乙开发的是B模块，那么乙应该在主分支的基础上新建一个分支b，在分支b上进行开发。这样就能够保证甲和乙两个人各自开发各自的模块，相互独立，各不影响。最后当两者同时开发完成后，将甲的分支a与乙的分支b合并到主分支上去。

    一般来说，当我们使用git init初始化一个git仓库的时候，git会默认帮我们生成一个主分支，也是我们默认所在的分支，一般情况下，master主分支上的代码不会被改变，如果某个人想要改变代码，必须要新建一个分支，然后在分支的基础上进行改动，测试没有问题后，在合并到主分支上去。

    上面我们已经说了查看分支的指令，如果我们想要新建一个分支，怎么办呢？这也是很简单的，执行如下命令就可以了新建一个分支。

    git branch hello

    执行上面的指令就新建了一个名为hello的分支，这个hello分支与master主分支的内容是完全一样的，不会发生任何变化，相当于从主分支克隆出来一个副本进行更改，如果确认没有错误了，在合并到主分支上去。这就相当于大学更改论文一样，我们一般会保留一个版本（master主分支），如果这个版本的论文需要更改，我们就复制出论文的一个副本（branch）出来更改，确定复制出来的副本（branch）论文内容没有错误了，我们就将副本（branch）的内容合并到保留版本（master主分支）上去。这样既可以保证主分支的内容始终都是正确的。

    现在我们已经新建了一个branch，我们在上面使用了git branch查看了当前的分支，的确时出现了两个分支，但是细心的同学估计已经看到了在master主分支的前面是有一个星号（*）的，这个星号表明现在我们仍然处在master上，那么如果我们要在分支上进行开发，我们应该怎样切换到其他分支上去呢？哈哈，当然，这也是不难的。执行下面的命令：

    git checkout hello

    现在再使用git branch 查看当前的分支，你会惊奇的发现，星号现在已经在hello的前面了，现在你就可以在当前分支下进行我们的开发了。

    这样先新建再切换还是有点麻烦，有一步解决问题的办法。执行如下指令：

    git checkout -b world

    这行命令的意思就是先新建一个world分支，然后再自动切换到world分支。

    既然有新建分支，肯定就有删除分支的操作。如果我们要删除hello分支，我们可以直接使用如下指令，删除hello分支的指令如下：

    git branch -d hello

    上面的这种情况只使用于里面什么都没有的情况，但是如果我们主分支中有代码，而且hello分支中也有代码，当我们执行删除的时候，我们是删除不了这个分支的，因为git会自作主张的提示你，你的代码没有和master分支合并，不建议你删除，此时你可以执行如下指令强制删除这个分支：

    git branch -D hello

    最后一个就是代码合并，代码合并有几点要求必须要注意，主要就是我们在合并代码的同时，我们必须先切换到master上，然后执行如下指令就可以把branch（hello）上的代码成功合并到主分支上去。

    git merge hello

    总结

    好了，关于git的本地操作基本上就讲完了，这也是我平时在开发当中用到的所有本地命令。其实内容也不算很多，都是很简单的命令，当你运用熟练了之后，你会爱上git命令行方式的操作的，相信我！好了，下面的一章我们将会讲解git远程仓库的一些常见操作。

  • 使用github + hexo搭建一个属于自己的博客详细教程

    2016-06-17

    什么是Hexo？

    Hexo是一个快速，简洁且高效的博客框架。主要优点就是利用MarkdownPad（或其他渲染引擎）解析文章，它主要是可以利用漂亮的主题生成静态网页。接下来就是用Hexo搭建博客的详细过程。有心的童鞋可以去hexo官网看看。其实官网的教程还不错，讲的挺详细的。

    首先我们说一下安装过程

    1.安装node.js
2.安装hexo
3.安装git

    由于hexo是基于node.js的，所以在开始之前必须先安装node.js，安装node.js的方法就太简单了。一路下一步下一步就可以了。至于如何去下载，请各位皇上或者太后移驾Node.js官网。

    至于安装git客户端，和如何注册github账号，不会的请自行百度。

    安装Hexo

    当node.js和git都安装好了，我们就可以安装hexo了。打开cmd，执行如下命令操作：

    npm install -g hexo

    -g 代表全局安装hexo，如果安装的时候选择了-g这个参数进行全局安装，那么也就意味着我们可以在命令行的任何地方使用hexo指令。

    初始化

    打开命令行，进入你想要创建blog文件的目录，这里我选择F盘根目录为博客目录（注意：执行如下命令后，会自动在你制定的目录下创建一个文件夹，我这里是创建的是F：/blog）

    hexo  init blog

    blog是你建立的文件夹的名称，用cd指令进入到该目录下，执行命令，安装hexo依赖：

    cd blog
hexo install

    接下来执行开启hexo服务命令

    hexo server

    当服务开启后，我们可以在命令行中看到如下结果

    此时我们打开浏览器，在地址栏输入http://localhost:4000/，回车，然后就可以看到hexo的hello world页面了。

    关联git

    本地设置好了之后，我们接下来开始关联Github.

    第一步：添加SSH key到github上（这个挺重要的，因为以后我们要使用github向远程服务器推送本地代码，也是需要配置ssh Key的，当然这是后话，我将会在接下来的git操作中讲到配置SSH key 以及推送本地代码到远程仓库）

    之前我已经让你们安装了git，所以此时你们一定是没有SSH key这个玩意的。如果没有就可以直接生成一个SSH key，如果有，那就找到它。检测本地是否存在SSH key 

    ls -al ~/.ssh

    这个指令是linux下的指令，在windows下是使用不了的，不过我们可以利用一个搜索工具在硬盘上检索是否存在这个key。搜索我推荐使用everything for search，这个搜索程序是很强大的一款搜索程序，几乎能够实时定位你要检索的文件的位置。要检查是否存在ssh key，直接在everything for search中键入.ssh 就可以了。

    生成SSH key的指令为

    $ ssh-keygen -t rsa -C "your_email@163.com"

    一般键入上面的生成SSH key的指令后，什么都不用管，直接连续三个回车就可以生成SSH_key 了，而且默认的生成路径在一般都在”C:\Users\Administrator.ssh”，进入这个文件夹，里面有id.rsa文件和id_rsa.pub,用记事本打开id_rsa.pub，里面的信息即为SSH key，将这些信息复制到GitHub上的Add SHH key就可以了。

    进入GitHub-->Settings-->SSH keys-->add SSH key

    进入后，Title字段是随便填写的，将复制的内容粘贴到key里面，点击Add key就可以了。对了，还要记得将”your_email@163.com”这个字符串替换成你自己注册github的邮箱账号就可以了。

    发布博客

    使用命令行进入blog目录，执行如下指令

    hexo new "postName"

    文件名为postName.md的文件会在新建在F:/blog/source/_posts目录下。你可以用你自己喜欢的文本编辑器编辑这个文件，我一般都是使用MarkdownPad来编辑这个种文件，主要是因为MarkdownPad带有实时预览的效果。

    文章编辑完成之后，使用命令行进入到blog目录下，执行如下命令来发布blog。

    hexo generate //生成静态页面
hexo deploy //将文章部署到github上

    走到这步，整个blog的搭建以及发布就完成了，下面介绍一下hexo主题的安装，添加评论功能以及绑定个人域名，如果前面你玩的不过瘾，那么你可以继续下去。

    安装主题

    hexo的主题你可以去官网下载自己喜欢的主题，这里这里以hexo-theme-indigo为例。
    使用命令行工具进入到blog目录，在命令行里执行如下命令

    git clone git@github.com:yscoder/hexo-theme-indigo.git themes/indigo   // 下载主题 
npm install hexo-renderer-less --save  // 下载依赖
npm install hexo-generator-feed --save  // 下载依赖

    进入blog修改_config.yml

    //主题
theme: indigo
//插件依赖
plugins： -hexo-renderer-less -hexo-generator-feed

    这里，主题就安装成功了，然后我们进入blog/themes/indigo目录下，打开_config.yml开始配置我们自己的内容，比如名称，描述，头像等可以去修改blog/_config.yml文件和blog/themes/indigo/_config.xml

    修改完之后，使用命令行进入到blog目录下执行如下命令（这也是每次部署新文章要执行的命令）

    hexo clean //清楚缓存文件（db.json）和已经生成的静态文件
hexo generate  //生成缓存和静态文件
hexo deploy  //重新部署到服务器

    然后在浏览器中打开网址：http://selfass.github.io(将selfass换成你自己的用户名)能看到我们刚刚修改的内容。

    到这里，我们的blog基本就算完成了，如果你还想你的blog逼格高一点，那么你可以给你的blog设置一个专属域名。接下来我们就设置专属域名.

    更换博客域名

    说到域名嘛，最牛逼的域名网站就属”狗他爹”了，官方网站可以直接去百度搜索，GoDaddy。官网就是https://sg.godaddy.com/zh/

    Github端
    在F：/blog/themes/landscape/source目录下新建文件名为：CNAME文件，注意，是没有后缀名的。直接将自己的域名如：selfass.com写入。

    使用命令行进入到blog目录下执行如下命令重新部署：

    hexo clean
hexo generate
hexo deploy

    注意：网上有很多人都说域名映射文件是在github上直接新建CNAME文件，但是如果是这样的话，我们每次执行hexo deploy命令后CNAME文件就消失了，因为本地没有这个文件。

    域名解析
    如果将域名指向一个域名，实现与被指向域名相同的访问效果，需要增加CNAME记录。进入你买的域名的网站，在你购买的域名后面点击：解析–>添加解析

    记录类型：CNAME

    主机记录：将域名解析为test.com（不带www）

    记录值：selfass.github.io.（不要忘记最后面的一个点， 将selfass改为你自己的用户名，点击保存就可以了）。

    此时，点击访问http：//selfass.com和访问http://selfass.github.io效果是相同的，到此大功告成。
