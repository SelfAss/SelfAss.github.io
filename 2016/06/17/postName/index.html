使用github + hexo搭建一个属于自己的博客详细教程

什么是Hexo？

Hexo是一个快速，简洁且搞笑的博客框架。主要优点就是利用MarkdownPad（或其他渲染引擎）解析文章，它主要是可以利用漂亮的主题生成静态网页。接下来就是用Hexo搭建博客的详细过程。有心的童鞋可以去hexo官网看看。其实官网的教程还不错，讲的挺详细的。

首先我们说一下安装过程

1.安装node.js
2.安装hexo
3.安装git

由于hexo是基于node.js的，所以在开始之前必须先安装node.js，安装node.js的方法就太简单了。一路下一步下一步就可以了。至于如何去下载，请各位皇上或者太后移驾Node.js官网。

至于安装git客户端，和如何注册github账号，不会的请自行百度。

安装Hexo

当node.js和git都安装好了，我们就可以安装hexo了。打开cmd，执行如下命令操作：

npm install -g hexo

-g 代表全局安装hexo，如果安装的时候选择了-g这个参数进行全局安装，那么也就意味着我们可以在命令行的任何地方使用hexo指令。

初始化

打开命令行，进入你想要创建blog文件的目录，这里我选择F盘根目录为博客目录（注意：执行如下命令后，会自动在你制定的目录下创建一个文件夹，我这里是创建的是F：/blog）

hexo  init blog

blog是你建立的文件夹的名称，用cd指令进入到该目录下，执行命令，安装hexo依赖：

cd blog
hexo install

接下来执行开启hexo服务命令

hexo server

当服务开启后，我们可以在命令行中看到如下结果

此时我们打开浏览器，在地址栏输入http://localhost:4000/，回车，然后就可以看到hexo的hello world页面了。

关联git

本地设置好了之后，我们接下来开始关联Github.

第一步：添加SSH key到github上（这个挺重要的，因为以后我们要使用github向远程服务器推送本地代码，也是需要配置ssh Key的，当然这是后话，我将会在接下来的git操作中讲到配置SSH key 以及推送本地代码到远程仓库）

之前我已经让你们安装了git，所以此时你们一定是没有SSH key这个玩意的。如果没有就可以直接生成一个SSH key，如果有，那就找到它。检测本地是否存在SSH key 

ls -al ~/.ssh

这个指令是linux下的指令，在windows下是使用不了的，不过我们可以利用一个搜索工具在硬盘上检索是否存在这个key。搜索我推荐使用everything for search，这个搜索程序是很强大的一款搜索程序，几乎能够实时定位你要检索的文件的位置。要检查是否存在ssh key，直接在everything for search中键入.ssh 就可以了。

生成SSH key的指令为

$ ssh-keygen -t rsa -C "your_email@163.com"

一般键入上面的生成SSH key的指令后，什么都不用管，直接连续三个回车就可以生成SSH_key 了，而且默认的生成路径在一般都在”C:\Users\Administrator.ssh”，进入这个文件夹，里面有id.rsa文件和id_rsa.pub,用记事本打开id_rsa.pub，里面的信息即为SSH key，将这些信息复制到GitHub上的Add SHH key就可以了。

进入GitHub-->Settings-->SSH keys-->add SSH key

进入后，Title字段是随便填写的，将复制的内容粘贴到key里面，点击Add key就可以了。对了，还要记得将”your_email@163.com”这个字符串替换成你自己注册github的邮箱账号就可以了。

第二步： 在GitHub上创建一个仓库，作为存放博客页面的仓库

登陆你的github账号，新建仓库，名称为”selfass.github.io”,建立过程如下图所示。

通过命令行的方式，我们进入到blog文件夹下，选择_config.yml,用文本编辑器打开，将该文件的最后修改成如图所示的样子
原本打开是这个样子

替换成这个样子

通过命令行的方式，我们定位在blog文件夹下，执行该命令生成静态页面

命令为：dexo generate

不出意外的话，应该出现这样的结果

但是window系统应该会出错的，所以如果你出错了就执行下面的一条命令

npm install hexo --save
（不出错就忽略这条指令，直接执行下面的指令）

最后再在blog目录下单击鼠标右键，选中gitbash，在gitbash里面执行以下命令

hexo deploy（你在gitbash里面执行这个命令的时候，可能会报另外一个错误。仔细看，如果有filename too long这个错误，这个错误的原因很简单，因为在window系统上，git文件的路径长度是有255字符限制的，如果文件字符过长，就会这个错误，解决办法如下
进入仓库目录下，打开git bash，执行如下指令：
git config core.longpaths true
如果想要给所有的仓库都设置，也可以执行 git config --global core.longpaths true)

执行结果

不出意外，你执行这条语句依然会报错，如果报错在cmd命令下进入blog文件夹下，执行如下命令

（老规矩，出错就执行，不出错就忽略这条语句，这条语句是在cmd进入blog文件夹中执行的。）

执行完后，再执行hexo generate 和 hexo deploy应该就没错了，此时浏览器中打开网址http://selfass.github.io（将selfass换成你的用户名）能看到和打开http://localhost:4000时一样的页面

发布博客

使用命令行进入blog目录，执行如下指令

hexo new "postName"

文件名为postName.md的文件会在新建在F:/blog/source/_posts目录下。你可以用你自己喜欢的文本编辑器编辑这个文件，我一般都是使用MarkdownPad来编辑这个种文件，主要是因为MarkdownPad带有实时预览的效果。

文章编辑完成之后，使用命令行进入到blog目录下，执行如下命令来发布blog。

hexo generate //生成静态页面
hexo deploy //将文章部署到github上

走到这步，整个blog的搭建以及发布就完成了，下面介绍一下hexo主题的安装，添加评论功能以及绑定个人域名，如果前面你玩的不过瘾，那么你可以继续下去。

安装主题

hexo的主题你可以去官网下载自己喜欢的主题，这里这里以hexo-theme-indigo为例。
使用命令行工具进入到blog目录，在命令行里执行如下命令

git clone git@github.com:yscoder/hexo-theme-indigo.git themes/indigo   // 下载主题 
npm install hexo-renderer-less --save  // 下载依赖
npm install hexo-generator-feed --save  // 下载依赖

进入blog修改_config.yml

//主题
theme: indigo
//插件依赖
plugins： -hexo-renderer-less -hexo-generator-feed

这里，主题就安装成功了，然后我们进入blog/themes/indigo目录下，打开_config.yml开始配置我们自己的内容，比如名称，描述，头像等可以去修改blog/_config.yml文件和blog/themes/indigo/_config.xml

修改完之后，使用命令行进入到blog目录下执行如下命令（这也是每次部署新文章要执行的命令）

hexo clean //清楚缓存文件（db.json）和已经生成的静态文件
hexo generate  //生成缓存和静态文件
hexo deploy  //重新部署到服务器

然后在浏览器中打开网址：http://selfass.github.io(将selfass换成你自己的用户名)能看到我们刚刚修改的内容。

到这里，我们的blog基本就算完成了，如果你还想你的blog逼格高一点，那么你可以给你的blog设置一个专属域名。接下来我们就设置专属域名.

更换博客域名

说到域名嘛，最牛逼的域名网站就属”狗他爹”了，官方网站可以直接去百度搜索，GoDaddy。官网就是https://sg.godaddy.com/zh/

Github端
在F：/blog/themes/landscape/source目录下新建文件名为：CNAME文件，注意，是没有后缀名的。直接将自己的域名如：selfass.com写入。

使用命令行进入到blog目录下执行如下命令重新部署：

hexo clean
hexo generate
hexo deploy

注意：网上有很多人都说域名映射文件是在github上直接新建CNAME文件，但是如果是这样的话，我们每次执行hexo deploy命令后CNAME文件就消失了，因为本地没有这个文件。

域名解析
如果将域名指向一个域名，实现与被指向域名相同的访问效果，需要增加CNAME记录。进入你买的域名的网站，在你购买的域名后面点击：解析–>添加解析

记录类型：CNAME

主机记录：将域名解析为test.com（不带www）

记录值：selfass.github.io.（不要忘记最后面的一个点， 将selfass改为你自己的用户名，点击保存就可以了）。

此时，点击访问http：//selfass.com和访问http://selfass.github.io效果是相同的，到此大功告成。
